---

## 1. 为什么需要 Dropout（动机）

* 深层网络**表达力强**，容易在训练集上“记忆”噪声与偶然相关 ⇒ **过拟合**。
* 期望好的模型对**小扰动**不敏感（**鲁棒/平滑**）。
* 思路：在训练时向**隐藏层**注入噪声，让网络学会**分散依赖**、打破“共适应性”（co-adaptation）。

---

## 2. 核心思想（直观）

* 在前向传播中，**按概率随机屏蔽**一部分神经元的输出（置 0）。
* 等价于每次用一个**子网络**训练，多个子网络共享参数；测试时使用**全网络**。

---

## 3. 数学定义（Inverted Dropout）

设隐藏层激活为 $h$，丢弃率为 $p$（保留率 $q=1-p$）：

* 采样掩码 $m\sim \text{Bernoulli}(q)$（与 $h$ 同形状）。
* 训练时用
  $$
  h'=\frac{m\odot h}{q}
  $$
  使得 $\mathbb{E}[h']=h$（**期望不变性**）。
* 测试时：**不做丢弃**，也**不再缩放**（因为训练期已缩放）。

> 注：这就是“**反向缩放**（Inverted Dropout）”的命名来源。

---

## 4. 训练 vs 测试

* **训练阶段**：随机屏蔽 + 除以 $q$（保持期望一致）。
* **测试阶段**：不屏蔽，不缩放；即用完整的网络做确定性前向。

---

## 5. 实现要点（以 PyTorch 为例）

### 5.1 自定义函数（演示原理）

```python
import torch

def dropout_layer(X, p: float):
    # 断言：确保丢弃率 p 在 [0, 1] 的有效范围内
    assert 0.0 <= p <= 1.0
    
    # 如果 p=1，表示所有神经元都被丢弃，直接返回全零张量
    if p == 1.0: 
        return torch.zeros_like(X)
    
    # 如果 p=0，表示没有神经元被丢弃，直接返回原始输入 X
    if p == 0.0: 
        return X
    
    # 1. 生成掩码：创建一个与 X 形状相同的随机张量（均匀分布在 [0, 1)）
    #    通过与 p 比较，生成一个布尔掩码（True/False）
    #    .float() 将布尔值转换为 0.0 和 1.0，其中 1.0 的概率是 1-p
    mask = (torch.rand_like(X) > p).float()
    
    # 2. 应用掩码并进行缩放（Inverted Dropout）
    #    mask * X：将部分激活置零
    #    / (1.0 - p)：将被保留的激活值放大，以保持其期望值在训练和测试期间一致
    return mask * X / (1.0 - p)
```

> 关键：**用均匀分布 `rand`** 采样掩码；**不要**用 `randn`（正态分布），否则阈值比较不是“按概率 p 丢弃”。

### 5.2 模块化用法

```python
import torch.nn as nn

# 定义一个包含 Dropout 的多层感知机
net = nn.Sequential(
    nn.Flatten(),
    # 第一个全连接层和激活函数
    nn.Linear(784, 256), 
    nn.ReLU(),
    # 在第一个隐藏层后添加 Dropout，丢弃率为 0.2
    # 靠近输入层的丢弃率通常设置得小一些
    nn.Dropout(p=0.2),
    
    # 第二个全连接层和激活函数
    nn.Linear(256, 256), 
    nn.ReLU(),
    # 在第二个隐藏层后添加 Dropout，丢弃率为 0.5
    # 靠近输出层的丢弃率可以设置得大一些
    nn.Dropout(p=0.5),
    
    # 输出层
    nn.Linear(256, 10),
)

# --- 训练与评估模式切换 ---

# 将模型设置为训练模式
# 在此模式下，Dropout 层会正常工作（随机丢弃神经元）
net.train()   

# ... 在这里执行模型训练代码 ...

# 将模型设置为评估（推理）模式
# 在此模式下，Dropout 层会被禁用，所有神经元都会被使用
net.eval()    
```

## 6. 超参数与实践建议

* **丢弃率 p**：常见选择

  * 输入附近：0.1~0.3
  * 深层/大层：0.3~0.6
* **何时使用**：模型容量较大、训练集较小或出现明显过拟合时。
* **与其它正则化**：通常与 **权重衰减（L2）**、**数据增强**、**早停**等联合使用更稳。
* **监控指标**：训练/验证损失与准确率曲线；注意过大 $p$ 可能**欠拟合**。

---

## 7. 与权重衰减（L2）的对比与组合

| 方面   | Dropout                  | 权重衰减（L2）          |
| ---- | ------------------------ | ----------------- |
| 作用对象 | **激活值**（前向时注入噪声）         | **参数**（目标函数加范数惩罚） |
| 直观效果 | 打破神经元共适应，像“模型集成”         | 限制参数幅度，偏向更“简单”的函数 |
| 计算开销 | 训练期略增                    | 忽略不计              |
| 组合   | **常见且有效**：两者互补，通常带来更好的泛化 |                   |

---

## 8. 变体与拓展

* **Dropout on inputs**：也可对输入做丢弃（相当于数据噪声）。
* **SpatialDropout/Dropout2d**：在卷积中按**通道**丢弃，保留空间结构。
* **MC Dropout**（推理时保留 Dropout，多次采样）：可用于**不确定性估计**。

---

## 9. 常见坑 & 调试清单

* **忘记 `model.eval()`**：推理时还在随机丢弃，导致结果抖动/变差。
* **没有使用 Inverted 方案**：训练期未除以 $(1-p)$，导致测试期需要额外缩放，容易出错。
* **`randn` vs `rand`**：掩码应来自 **均匀分布阈值比较** 或直接 Bernoulli 采样。
* **丢弃率过大**：训练损失下降困难、验证表现差 ⇒ 适当减小 $p$。
* **层位选择**：通常**激活函数之后**做 Dropout；对**批归一化后**再接 Dropout 的效果需按任务验证。

## 10. 典型 Q&A（结合课程讨论要点）

* **Q：为什么要除以 $(1-p)$？**
  A：保持 $\mathbb{E}[h']=h$。训练和测试的激活量级一致，避免测试期再缩放。

* **Q：被丢弃的神经元权重本轮会更新吗？**
  A：不会。对应激活为 0，反向梯度为 0，权重无梯度更新。

* **Q：如何在验证/测试时关闭 Dropout？**
  A：调用 `model.eval()`（或 Keras `training=False`）；框架会自动关闭。

* **Q：为什么用 `rand` 而不是 `randn`？**
  A：`rand`∈[0,1] 均匀分布，`> p` 的概率正好是 $(1-p)$。`randn` 有负数且不是区间均匀，不对应所需概率。

* **Q：交换不同层的丢弃率会怎样？**
  A：通常**靠近输入层用更小的 p** 更稳；把高丢弃率放在浅层可能损伤特征提取，需用验证集比较。

## 11. 实验建议（可复现的思路）

1. **p 网格搜索**：固定结构，分别设 $(0.1,0.3), (0.2,0.5), (0.3,0.5)$ 比较验证集性能。
2. **与 L2 对比/组合**：仅 Dropout、仅 L2、两者并用，画学习曲线与最终准确率。
3. **方差跟踪**：记录各隐藏层激活在训练过程中的**方差变化**（有/无 Dropout），观察正则化效果。

---

## 12. 一句话总结

**Dropout = 训练期对隐藏激活做随机屏蔽 + 期望保持的缩放**；
它像“参数共享的子模型集成”，简单有效、常与 L2/数据增强联用，是缓解过拟合的默认利器。
