# 卷积层中的填充（Padding）和步幅（Stride） — 笔记

## 1. 为什么需要填充和步幅？

在卷积神经网络中，卷积操作会天然导致**输出尺寸不断缩小**，尤其是卷积核较大、层数较深时，这种缩小非常明显。

### 例子：

* 输入：32×32 图像
* 卷积核：5×5，步幅 = 1，填充 = 0

输出尺寸：

```
32 - 5 + 1 = 28  →  28×28
```

如果连续做 7 次：尺寸会逐步缩小到：

```
32 → 28 → 24 → ... → 4
```

这将导致图像“太快变小”，无法继续卷积。

所以我们需要：

* **Padding（填充）**：控制输出不变小或减少变小速度
* **Stride（步幅）**：控制输出缩小的速度

---

## 2. 填充 Padding

### 2.1 Padding 的直观含义：

在输入张量周围“补一些额外的行/列”，通常补 0，用于：

* 防止图像太快变小
* 保留边缘信息（边缘的像素在未填充时使用次数比中间少）

### 2.2 无填充与有填充的示例

假设输入为 3×3，卷积核为 3×3：

* **无填充**：只能卷一次 → 输出 1×1
* **Padding = 1**：输入变成 5×5，可以卷更多次，输出为 3×3

---

## 3. 输出尺寸公式（带 Padding 和 Stride）

输入尺寸：$(n_h \times n_w)$
卷积核尺寸：$(k_h \times k_w)$
padding：$(p_h, p_w)$
stride：$(s_h, s_w)$

输出高度 $H_{out}$ 和宽度 $W_{out}$ 的计算公式如下：

$$
H_{out} = \lfloor \frac{n_h - k_h + 2p_h}{s_h} \rfloor + 1
$$
$$
W_{out} = \lfloor \frac{n_w - k_w + 2p_w}{s_w} \rfloor + 1
$$

其中：
*   $(p_h, p_w)$ 是指在高度和宽度上的**单边填充**（例如，上下各填充 $p_h$ 行，左右各填充 $p_w$ 列）。
*   如果 stride 不能整除，结果**向下取整**（$\lfloor \cdot \rfloor$ 符号表示）。

---

## 4. Padding 的两种典型模式

### 4.1 SAME padding（输出尺寸保持不变）

当 stride = 1 时：

```
输出尺寸 = 输入尺寸
```

要求：

```
p = (k - 1)/2
```

卷积核通常为奇数大小（如 3、5、7），保证 p 为整数。

### 4.2 VALID padding（不填充）

即 p = 0，输出自然会缩小。

---

## 5. 步幅 Stride

### 5.1 Stride 的直观理解

步幅决定“卷积核每次移动多少格”。

* stride = 1：提取细粒度信息（移动一步）
* stride = 2：下采样（移动两步）

### 5.2 Stride 的效果

输入：6×6，卷积核：3×3

* stride = 1：输出为 4×4
* stride = 2：输出为 2×2

步幅可以显著减小输出大小，因此常用于：

* 图像下采样
* 加快模型计算速度

---

## 6. 填充 + 步幅：一起控制输出

最终输出大小取决于两者共同作用。

### 示例：

输入 32×32，核 5×5，padding=2，stride=1

$$
H_{out} = \lfloor \frac{32 - 5 + 2 \times 2}{1} \rfloor + 1 = 31 + 1 = 32
$$

→ SAME 卷积：输出仍然是 32×32

---

## 7. 为什么图像会经卷积不断变小？

卷积本质是：

* 每次只对“可以完整放下卷积核的区域”计算输出

当不填充时：

* 边缘无法完整覆盖 → 边缘被丢弃

这就是：

```
(n - k + 1)
```

的来源。

通过 padding 补齐，可以保持尺寸或者减少缩小速度。

---

## 8. 小结

### Padding 的作用：

* 防止尺寸过快缩小
* 保留边缘特征
* 可实现 SAME 卷积

### Stride 的作用：

* 控制输出缩小速度
* 可以快速下采样

### 输出尺寸公式：

$$
H_{out} = \lfloor \frac{n_h - k_h + 2p_h}{s_h} \rfloor + 1
$$
$$
W_{out} = \lfloor \frac{n_w - k_w + 2p_w}{s_w} \rfloor + 1
$$

Padding 与 Stride 是卷积层中控制输出形状的核心机制，理解它们可以帮助你：

* 设计网络结构
* 推导输出尺寸
* 解决卷积层尺寸过小的问题
